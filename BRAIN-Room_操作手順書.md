# システム概要

BRAIN-Roomは、複数のAIキャラクターを設定し、指定したテーマについてリアルタイム議論を行わせることができるブレインストーミングシステムです。

## 主な機能
- カスタムキャラクター作成：性格・話し方・背景を詳細設定
- プリセットセット：すぐに使える定義済みキャラクターグループ
- リアルタイム議論：テーマに基づく自然な対話形式の討論
- 自動タグ付け：発言の種類（同意・問題提起・重要情報など）を自動分類
- 多角的可視化：チャット・マインドマップ・ノードリスト・群島ビューで議論を分析

## 4つの可視化モード
| モード | 説明 | 用途 |
|--------|------|------|
| チャットビュー | 時系列の会話形式表示 | 議論の流れを確認 |
| マインドマップビュー | 階層構造での関連性表示 | トピックの関係性を把握 |
| ノードリストビュー | 議論要素の一覧表示 | ポイントの整理と確認 |
| 群島ビュー | 島嶼形式のインタラクティブ表示 | 視覚的な議論マップ探索 |

# システムへのアクセス

## ステップ1: ブラウザでアクセス
1. 推奨ブラウザを開きます（Chrome、Firefox、Safari、Edge最新版）
2. アドレスバーに以下のURLを入力：
   ```
   https://v0-ai-chat-system-prototype.vercel.app/
   ```

## ステップ2: パスワード認証
システムがロードされると、認証画面が表示されます。

1. パスワード入力欄に以下のパスワードを正確に入力：
   ```
   X|&fqa%MyXr7
   ```
   注意: 大文字・小文字、記号を正確に入力してください

2. [Unlock]ボタンをクリックまたはEnterキーを押す

## アクセス成功の確認
認証が成功すると、BRAIN-Roomのメイン画面が表示され、左上に「BRAIN-Room」というタイトルが表示されます。

# キャラクターセットの管理

## プリセットセットの利用（推奨）

初心者はプリセットセットから始めることをお勧めします。

### ステップ1: プリセットの選択
1. 左パネルからプリセットセットをクリック
   - 「友人グループ」: カジュアルな議論・ブレインストーミング向け
   - 「職場の同僚」: ビジネス議論・プロジェクト検討向け

### ステップ2: キャラクター確認
右パネルで各キャラクターの設定を確認：
- Jim: 楽観的で前向き、新しいことに挑戦するのが好き
- Sarah: 慎重で分析的、物事を合理的に考える
- Mike: のんびりして平和主義、ユーモアがある
- Emma: 情熱的で感情豊か、直感を大切にし人の気持ちに敏感

### ステップ3: 議論開始
「このセットで開始」ボタンをクリックして議論画面へ移動

## カスタムセットの作成

独自のキャラクターセットを作成したい場合：

### ステップ1: 新規作成の開始
1. 左パネルの「新規作成」をクリック
2. 新しい編集画面が表示されます

### ステップ2: セット名の設定
1. 「セット名」欄に分かりやすい名前を入力
   - 例：「マーケティング戦略会議」「技術検討チーム」「創作アイデア会議」

### ステップ3: 各キャラクターの詳細設定
各キャラクタータブ（Jim、Sarah、Mike、Emma）をクリックして設定：

1. 名前
   例：「田中マーケター」「山田エンジニア」「佐藤デザイナー」

2. 人格・個性（重要）
   例：
   - 「論理的で分析的な思考を好む。データに基づいた判断を重視」
   - 「創造的でアイデア豊富。既成概念にとらわれない発想力」
   - 「慎重で現実的。リスク評価と実現可能性を重視」

3. 話し方
   例：
   - 「です・ます調で丁寧。専門用語を使いがち」
   - 「カジュアルで親しみやすい。比喩表現を多用」
   - 「簡潔で要点を突く。質問形式で話すことが多い」

4. 背景
   例：
   - 「10年のマーケティング経験。特にデジタル分野に詳しい」
   - 「スタートアップでの開発経験豊富。新技術への関心が高い」
   - 「大手企業での管理職経験。チームマネジメントが得意」

### ステップ4: 保存と開始
設定完了後、「このセットで開始」をクリックして保存・議論開始

## 既存セットの編集

### 編集手順
1. 左パネルから編集したいセットをクリック
2. 右パネルで内容を修正

# 議論の実施

キャラクターセットを選択後、議論メイン画面が表示されます。

## 議論画面のレイアウト

上部に現在のキャラクター一覧、下部にプロジェクト設定エリアが配置されています。

## ステップ1: 議論テーマの設定

### テーマ入力のコツ
議論テーマを入力欄に記入します。具体的で議論しやすいテーマを選ぶのがポイントです。

#### 良いテーマ例：
- 「チームワークを向上させるためのAIツール導入について」
- 「新商品のマーケティング戦略：ターゲット層と販促方法」
- 「リモートワークとオフィス勤務のバランス改善案」
- 「SDGsを意識した企業活動の具体的施策」
- 「顧客満足度向上のためのサービス改善点」

#### 避けるべきテーマ例：
- 「良いアイデア」（抽象的すぎる）
- 「どうすればいい？」（範囲が不明確）
- 「はい・いいえ」で答えられる質問

## ステップ2: 議論の開始

1. テーマ入力後、「会議を開始」ボタンをクリック
2. ボタンが「会議中...」に変わり、議論が開始されます
3. 「停止」ボタンが表示され、必要時に議論を停止可能

## ステップ3: リアルタイム議論の確認

議論が開始されると、画面下部に4つのビュータブが表示されます：

[チャットビュー] [マインドマップビュー] [ノードリストビュー] [群島ビュー]

### チャットビュー（メイン）
キャラクターたちのリアルタイム会議が表示されます。各発言にはキャラクター名とアイコンが表示され、発言の種類が自動的にタグ付けされます。

#### 自動タグ機能
各発言には以下のタグが自動付与されます：
- 気づき: 新しい観点や発見
- 問題提起: 課題や懸念点の指摘
- 合意: 他の意見への賛同
- 重要情報: キーとなる情報やデータ
- まとめ: 議論の整理や要約

## 議論中の操作

### 一時停止・再開
- 「停止」ボタン: 議論を一時停止
- 議論内容を整理したいときに使用

### ビュー切り替え
- 議論中でもタブを切り替えて多角的に確認可能
- リアルタイムで内容が更新されます

# 議論結果の確認と可視化

BRAIN-Roomの最大の特徴は、議論を4つの異なる視点で可視化できることです。

## 4つの可視化モード詳細

### 1. チャットビュー（基本ビュー）
用途: 議論の流れと文脈を時系列で確認

特徴:
- 発言の順序とタイミングが分かる
- 自動タグ付けで発言の種類を分類
- キャラクター毎の発言を色分け表示
- リアルタイム更新で議論の進行を追跡

活用シーン: 
- 議論の全体的な流れを把握したい
- 特定の発言への反応を確認したい
- 議事録として使用したい

---

### 2. マインドマップビュー
用途: トピック間の階層関係と関連性を視覚化

特徴:
- 中心テーマから枝分かれする構造表示
- 関連するアイデアのグループ化
- ノードクリックで詳細確認可能
- 議論の論理構造を整理

活用シーン:
- アイデアの関連性を整理したい
- 議論の構造を理解したい
- プレゼンテーション資料の作成準備

---

### 3. ノードリストビュー
用途: 議論で出た要素を項目別に整理して一覧表示

特徴:
- 議論された全ポイントを漏れなく表示
- カテゴリ別（問題・解決策・メリット等）に分類
- 各項目の詳細説明と関連発言へのリンク
- チェックリスト形式で進捗管理に活用可能

活用シーン:
- 議論のポイントを整理したい
- アクションアイテムを抽出したい
- 次回議論の参考資料として使用

---

### 4. 群島ビュー（アーキペラゴビュー）
用途: トピックを島として表現し、関連性を海流で視覚化

特徴:
- 各トピックを独立した「島」として表示
- 関連性の強さを島間の距離や海流で表現
- インタラクティブな探索が可能
- 直感的で美しい可視化

**島間距離の詳細仕様:**
- **基本配置**: 各島は画面上のXY座標（0〜100のパーセンテージ値）で配置
- **距離計算**: ユークリッド距離（直線距離）で計算
- **接続判定**: 島間の距離から両島の半径を引いた値が、画面対角線長の一定比率（レイヤーにより1.5%〜6%）以下の場合に接続線を描画
- **レイヤー構造**:
  - 最外層（レイヤー4）: 対角線長の6%以下で接続（薄い色、広範囲の関連性）
  - 中間層（レイヤー3）: 対角線長の4.5%以下で接続
  - 内側層（レイヤー2）: 対角線長の3%以下で接続
  - 最内層（レイヤー1）: 対角線長の1.5%以下で接続（濃い色、強い関連性）
- **視覚表現**: 
  - 近い島同士は太い線で接続され、強い関連性を示す
  - 遠い島同士でも一定距離内なら薄い線で接続され、弱い関連性を示す
  - 島のサイズ（重要度）も接続判定に影響（大きな島ほど遠くまで接続可能）
- **インタラクション**:
  - ズーム・パン機能で詳細表示が可能
  - ホバー時に島が微妙に拡大（5%）してハイライト
  - クリックで島の詳細情報を表示

活用シーン:
- 複雑な議論の全体像を俯瞰したい
- 関連性の発見やパターン認識
- クリエイティブなアイデア発想の支援

## ビュー切り替えの方法

議論開始後、画面下部のタブで簡単に切り替え可能：

[チャットビュー] [マインドマップビュー] [ノードリストビュー] [群島ビュー]

## 効果的な活用の流れ

1. チャットビュー: まず議論の全体的な流れを把握
2. マインドマップ: 論理構造と関連性を理解
3. ノードリスト: 具体的なポイントと行動項目を整理
4. 群島ビュー: 新しい関連性や洞察を発見

# データの保存とエクスポート

## 議論結果の保存

現在のシステムでは、議論内容はセッション中のみ保持されます。重要な議論内容を保持したい場合は、以下の方法を推奨します：

### 推奨保存方法
1. ブラウザのスクリーンショット機能を使用
2. 各ビューで重要な部分をキャプチャ
3. チャットビューの内容をテキストとしてコピー・ペースト
4. マインドマップやノードリストの内容を手動で記録

## 新しい議論の開始

### 設定画面に戻る
1. 議論を停止した後、ブラウザを更新
2. キャラクターセット選択画面に戻ります
3. 同じセットまたは新しいセットで別のテーマについて議論可能

# よくある質問とトラブルシューティング

## よくある問題と対処法

### 議論が開始されない
**症状**: 「会議を開始」ボタンをクリックしても何も起こらない

**対処法**:
1. インターネット接続を確認
2. ブラウザを更新（F5キー）
3. テーマが入力されているか確認
4. ブラウザのJavaScriptが有効になっているか確認

### 議論が途中で止まる
**症状**: 議論が途中で進行しなくなる

**対処法**:
1. 「停止」ボタンをクリックして一度停止
2. 数秒待ってから再度「会議を開始」をクリック
3. ブラウザを更新して最初からやり直し

### キャラクターセットが保存されない
**症状**: カスタムキャラクターセットの設定が消える

**対処法**:
1. 必須項目（名前、人格・個性、話し方、背景）がすべて入力されているか確認
2. 各項目に十分な文字数が入力されているか確認
3. ブラウザのローカルストレージが有効か確認

### 画面表示が崩れる
**症状**: レイアウトが正しく表示されない

**対処法**:
1. 推奨ブラウザ（Chrome、Firefox、Safari、Edge最新版）を使用
2. ブラウザのズーム設定を100%に設定
3. ブラウザのキャッシュをクリア
4. ブラウザの拡張機能を一時的に無効化

### ビュー切り替えができない
**症状**: タブをクリックしてもビューが切り替わらない

**対処法**:
1. 議論が完全に開始されるまで待つ
2. ページを更新して再度試行
3. 別のブラウザで試行

## 推奨環境

### ブラウザ
- Google Chrome（最新版）
- Mozilla Firefox（最新版）
- Safari（最新版）
- Microsoft Edge（最新版）

### 端末仕様
- RAM: 4GB以上推奨
- 画面解像度: 1280x720以上推奨
- インターネット接続: 安定したブロードバンド接続

## パフォーマンス最適化のコツ

1. **他のタブを閉じる**: メモリ使用量を削減
2. **バックグラウンドアプリを最小限に**: CPUリソースを確保
3. **定期的な休憩**: 長時間の使用時は定期的にブラウザを再起動

## 注意事項

1. **データの保存**: キャラクターセットはブラウザのローカルストレージに保存されます
2. **プライバシー**: 議論内容は一時的にサーバーで処理されますが、長期保存はされません
3. **同時利用**: 複数のブラウザタブでの同時利用は推奨されません
4. **文字数制限**: 各入力欄には文字数制限があります（テーマ: 200文字、その他設定項目: 500文字程度）
